Solution overview

FlowLens – AI-powered debugging context for MCP agents

Magentic is an AI agents lab focused on building multi-agentic systems for the software world. FlowLens gives your AI coding agents complete browser context so they can reproduce issues, identify root causes, and propose fixes without manual log gathering or prompt engineering.

Workflow

Record · Share · Fix

Click record and reproduce the bug. FlowLens captures screen video, console output, network traffic, user interactions, storage state, navigation events, and system information automatically—no right-click inspect or manual screenshots.

  • 1-minute setup via browser extension and FlowLens MCP server
  • One shareable link per bug for teammates and AI agents
  • SPA-friendly support for History API and hash-based routing

Google Cloud

How FlowLens runs on Google Cloud

FlowLens is designed to live natively on Google Cloud, using managed services for ingest, storage, indexing, AI reasoning, and observability. This keeps the platform scalable, secure, and simple to operate for both Magentic and customers.

Data ingestion & storage

Browser recordings are streamed to Google Cloud via secure endpoints. Raw assets and structured events are stored in Cloud Storage, with metadata and session indexes maintained in BigQuery for fast retrieval and analytics.

  • Versioned, encrypted object storage for videos and traces
  • BigQuery schemas for sessions, requests, console events, and user actions
  • Support for retention policies per workspace
Cloud Storage BigQuery

AI reasoning & MCP interface

The FlowLens MCP API runs on Cloud Run, exposing REST and MCP endpoints that orchestrate calls to Vertex AI models. Agents can request summaries, root-cause hypotheses, reproduction steps, and patch suggestions generated from the captured flow.

  • Stateless container workloads auto-scale on Cloud Run
  • Vertex AI prompts tuned for debugging tasks
  • Optional integration with IDE or CI agents through MCP
Cloud Run Vertex AI

Security, privacy & compliance

FlowLens is privacy-first. Sensitive fields are redacted locally before upload, and all data in Google Cloud is encrypted with Cloud KMS–managed keys. Cloud IAM enforces workspace-level access control, while Cloud Logging and Cloud Monitoring provide full observability and audit trails.

  • End-to-end TLS and AES-256 encryption at rest
  • Fine-grained IAM roles for tenants and agents
  • Logging and metrics for security, performance, and cost
Cloud IAM Cloud KMS Cloud Logging & Monitoring
Architecture & usage

From browser session to AI-generated fix

FlowLens slots into existing development workflows with minimal friction. It acts as a capture and context layer that feeds AI agents, not as yet another ticketing system.

High-level architecture

The recording extension and MCP server collect session data and send it to Google Cloud. A lightweight ingest service persists events and media into Cloud Storage and BigQuery. MCP endpoints on Cloud Run fetch context and orchestrate Vertex AI calls for analysis.

  • Client: FlowLens extension + local MCP server
  • Backend: Cloud Run APIs, Cloud Functions for async tasks
  • Data: Cloud Storage, BigQuery, Logging, Monitoring
Scalable by design Serverless

Typical debugging flow

A developer records a failing user journey and shares the FlowLens link. An AI coding agent pulls the associated context via MCP on Cloud Run, analyzes network, console and DOM state with Vertex AI, and proposes code changes or test cases back to the team.

  • Better bug reports for distributed teams
  • Fewer context-switches between tools
  • Grounded AI suggestions tied to real sessions
